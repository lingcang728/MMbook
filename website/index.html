关于统一阅读这件事

知乎长文、播客笔记、本地 Markdown，
不必再在三个工具之间来回跳。

当你滑过这一行，世界为它静下来 ——
这是继承自 MMbook 的聚光灯模式，让此刻读到的段落自然亮起。

书架、进度、工具链，
都在同一处。
你只是，在读。

const library = [zhihu, podcast, markdown];

三合一，仍然只为阅读服务

不是堆功能，是把原本分散的工作流收进一座本地书库。

📖

Markdown 精读

继承 MMbook：聚光灯专注、主题、搜索、目录、段落编辑。单文件打开、外部变更重载，专注长文。

🎙️

播客转写

接入原 PodcastTranscriber：音频/视频进，中英双语 Markdown 出。角色分离、本地模型或云端 API。

🧠

知乎归档

原 Zhihu Packer 能力整合进桌面端：归档长文，写入统一书库，随时回到精读或连读。

📚

统一本地书库

知乎 / 手动导入 / 其他来源分目录保存，manifest + 阅读进度同在本机，一份备份带走全部书目。

🌐

精读 + 连读

精读在桌面端沉浸完成；连读可在系统浏览器按书目顺序阅读，并与桌面端共享进度。

🔒

隐私优先

正文、图片、进度、登录态与转写配置均在本机。浏览器连读只绑定本地回环地址，退出即停。

一座书库，三种入口

从书架顶部直接启动工具；临时打开不会污染长期书库，播客默认阅后即焚，主动导入才入库。
